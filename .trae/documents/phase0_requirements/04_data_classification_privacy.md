# 数据分类与隐私分级表

## 1. 文档目标

本文定义心理咨询个案助手 App MVP 阶段的数据分类、隐私分级、采集原则、存储与传输要求、日志限制、AI 使用边界和通知展示规则。

本表应作为后续产品设计、Flutter 客户端、后端 API、数据库、AI 调用、日志监控和测试验收的共同约束。

## 2. 分级定义

| 等级 | 名称 | 定义 | 示例 |
| --- | --- | --- | --- |
| P0 | 公开/低敏数据 | 不直接关联个人或个案，泄露影响较低 | App 版本、功能配置、通用模板名称 |
| P1 | 内部业务数据 | 与业务流程相关，但不含具体个案敏感内容 | AI 任务状态、页面配置、匿名统计 |
| P2 | 个人敏感数据 | 可识别咨询师或来访者，或与个案相关的基础信息 | 咨询师账号、个案编号、标签、年龄段 |
| P3 | 高敏心理健康数据 | 音频、转写文本、咨询原文、风险线索、报告、心理状态等高度敏感内容 | 录音文件、转写文本、咨询记录、风险提示、报告草稿、证据片段 |

## 3. 总体处理原则

1. **最小必要**：只采集完成 MVP 闭环所需的数据。
2. **默认敏感**：个案相关数据默认按 P2 或 P3 处理。
3. **去真实身份化**：MVP 不强制采集来访者真实姓名、身份证、手机号。
4. **传输加密**：所有服务端通信必须使用 HTTPS。
5. **安全存储**：token 使用客户端安全存储，服务端敏感字段按生产安全要求加密或隔离。
6. **日志脱敏**：普通日志不得记录完整咨询原文、风险证据和报告内容。
7. **AI 可追溯**：进入 AI 的内容需要记录任务元数据，但不在普通日志中落原文。
8. **人工确认**：AI 输出和风险提示未经确认不得成为正式记录。

## 4. 数据分类表

| 数据类别 | 字段/内容示例 | 分级 | 是否必采 | 存储位置 | 是否可进普通日志 | 是否可进 AI | 通知中是否可展示 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| App 基础信息 | app_version、platform、device_os | P0 | 是 | 客户端/服务端 | 是 | 否 | 否 |
| 功能配置 | 记录模板、风险等级配置 | P0/P1 | 是 | 服务端/客户端缓存 | 是 | 否 | 否 |
| 咨询师账号 | user_id、手机号/邮箱、姓名 | P2 | 是 | 服务端、客户端安全存储部分凭据 | 脱敏后可记录 | 否 | 否 |
| 登录凭据 | access token、refresh token | P2 | 是 | 客户端安全存储、服务端会话 | 否 | 否 | 否 |
| 个案基础信息 | case_id、昵称/编号、年龄段、性别 | P2 | 是 | 服务端、客户端缓存 | 仅 ID 可记录 | 可按最小必要进入 | 默认否 |
| 个案备注 | 主诉、背景、标签、咨询目标 | P3 | 否 | 服务端、客户端缓存 | 否 | 是，需任务化处理 | 否 |
| 录音文件 | App 内录音或上传的音频文件 | P3 | 是 | 对象存储、客户端临时缓存 | 否 | 进入转写 API，不直接进入总结模型 | 否 |
| 音频元数据 | duration、format、size、upload_status | P1/P2 | 是 | 服务端、客户端缓存 | 可记录，不含文件名敏感信息 | 否 | 否 |
| 转写任务 | transcription_job_id、provider、status、error_code | P1/P2 | 是 | 服务端 | 可记录，不含转写正文 | 否 | 否 |
| 转写文本 | 云端 API 输出的咨询文本 | P3 | 是 | 服务端、客户端缓存 | 否 | 是，核心输入 | 否 |
| 咨询记录原文 | 咨询师输入或编辑后的长文本记录 | P3 | 是 | 服务端、客户端草稿缓存 | 否 | 是，核心输入 | 否 |
| 结构化字段 | 情绪状态、干预方式、下次计划 | P3 | 否 | 服务端、客户端缓存 | 否 | 是 | 否 |
| AI 任务元数据 | job_id、job_type、status、model、prompt_version | P1/P2 | 是 | 服务端 | 可记录，不含原文 | 否 | 否 |
| AI 输入快照 | 进入模型的脱敏/原始任务输入 | P3 | 是 | 服务端受控存储 | 否 | 是 | 否 |
| AI 结构化输出 | SOAP/DAP 摘要、跟进建议 | P3 | 是 | 服务端、客户端缓存 | 否 | 可作为报告输入 | 否 |
| 风险提示 | 风险等级、风险类型、说明 | P3 | 是 | 服务端、客户端缓存 | 否 | 可作为报告输入但需确认 | 默认否 |
| 风险证据片段 | 触发风险提示的原文片段 | P3 | 是 | 服务端、客户端缓存 | 否 | 是 | 否 |
| 人工确认记录 | 确认/调整/误报、备注 | P3 | 是 | 服务端 | 否 | 可用于后续质量分析，需脱敏 | 否 |
| 报告草稿 | 阶段报告、咨询概况、后续建议 | P3 | 否 | 服务端、客户端缓存 | 否 | 可作为后续报告输入 | 否 |
| 导出文件 | PDF、复制文本、附件 | P3 | 否 | 对象存储/本地临时文件 | 否 | 否 | 否 |
| 提醒 | 标题、时间、状态、关联个案 | P2/P3 | 否 | 服务端、客户端缓存 | 仅 ID/状态可记录 | 否 | 仅展示低敏标题 |
| 操作审计 | 用户、动作、目标类型、时间 | P1/P2 | 是 | 服务端 | 可记录，不含敏感正文 | 否 | 否 |
| 崩溃日志 | 堆栈、设备、页面路径 | P1 | 是 | 监控平台 | 可记录，需过滤参数 | 否 | 否 |
| 用户反馈 | 对 AI 输出的评价、问题描述 | P2/P3 | 否 | 服务端 | 摘要可记录，正文需脱敏 | 可用于评估，需脱敏 | 否 |

## 5. AI 调用数据边界

### 5.1 可进入 AI 的内容

MVP 中可作为 AI 输入的数据：

- 转写文本。
- 咨询记录原文。
- 咨询师填写的结构化字段。
- 个案非真实身份标识，如 case_id、昵称或编号。
- 历史摘要，前提是用于生成阶段报告。
- 已确认的风险提示，前提是用于生成报告草稿。

### 5.2 不建议进入 AI 的内容

MVP 默认不应传入：

- 来访者真实姓名。
- 身份证号。
- 完整手机号。
- 家庭住址。
- 付款信息。
- 与当前任务无关的历史记录全文。
- 原始音频文件不应直接进入总结/风险模型，除非该模型明确承担语音理解任务且通过隐私评审。

### 5.3 AI 任务记录要求

每次 AI 调用必须记录：

- `ai_job_id`
- `job_type`
- `input_scope`
- `model_provider`
- `model_name`
- `prompt_version`
- `schema_version`
- `created_at`
- `status`
- `confirmation_status`

每次云端转写调用必须记录：

- `transcription_job_id`
- `audio_asset_id`
- `provider`
- `api_version`
- `audio_format`
- `duration_seconds`
- `created_at`
- `status`
- `error_code`

普通日志只能记录任务 ID、状态和耗时，不记录 P3 原文。

## 6. 本地存储规则

| 数据 | 客户端处理规则 |
| --- | --- |
| access token / refresh token | 使用 `flutter_secure_storage` |
| 录音临时文件 | 保存到 App 私有目录，上传完成后按策略清理或保留 |
| 咨询记录草稿 | 可本地加密缓存，提交成功后同步服务端 |
| 个案列表缓存 | 可缓存最小字段，避免缓存完整 P3 正文 |
| 转写文本和 AI 输出结果 | 可短期缓存，需支持退出登录后清除 |
| 导出文件 | 只在用户主动导出时生成，临时文件需可清理 |

## 7. 服务端存储规则

| 数据 | 服务端处理规则 |
| --- | --- |
| 用户账号 | 分表或字段级权限控制，敏感字段加密或脱敏展示 |
| 个案信息 | 按 user_id 隔离，禁止越权访问 |
| 录音文件 | 私有对象存储，禁止公开 URL，访问需鉴权和短期签名 |
| 转写文本 | P3 数据，按咨询记录同等级保护 |
| 咨询记录 | P3 数据，生产环境建议字段加密或加密表空间 |
| AI 输入输出 | 独立任务表或受控存储，限制日志和调试访问 |
| 风险提示 | 与记录关联，必须保留人工处理状态 |
| 审计日志 | 不保存敏感正文，仅保存动作、对象 ID、时间、IP/设备摘要 |

## 8. 日志与监控规则

### 8.1 允许记录

- 请求 ID。
- 用户 ID 的内部标识。
- API 路径。
- 状态码。
- 耗时。
- 音频上传任务 ID。
- 转写任务 ID。
- AI 任务 ID。
- 错误码。
- 客户端版本。

### 8.2 禁止记录

- 咨询记录原文。
- 录音文件内容。
- 转写文本。
- 风险证据片段。
- 报告正文。
- token。
- 验证码。
- 真实身份信息。
- 任何未脱敏 P3 字段。

### 8.3 错误上报过滤

客户端错误上报需过滤：

- 输入框内容。
- 路由 query 中的敏感参数。
- 本地缓存内容。
- API 响应中的 P3 字段。

## 9. 通知展示规则

| 通知类型 | 锁屏/系统通知展示 | App 内展示 |
| --- | --- | --- |
| 普通提醒 | “你有一条待处理提醒” | 可展示提醒标题 |
| 个案跟进 | “有个案需要跟进” | 可展示个案昵称/编号和提醒内容 |
| 中高风险跟进 | “有重要提醒待处理” | 进入 App 后展示风险相关内容 |
| 转写任务完成 | “录音转写已完成” | 可展示关联个案昵称/编号和任务状态 |
| AI 任务完成 | “AI 整理已完成” | 可展示任务类型和关联记录 |
| 报告生成完成 | “报告草稿已生成” | 可展示报告标题 |

默认系统通知不展示录音内容、转写原文、咨询原文、风险证据、报告正文。

## 10. 权限与访问控制

MVP 最小权限模型：

- 咨询师只能访问自己创建的个案和记录。
- API 必须校验 `user_id` 与资源归属关系。
- 管理员后台不在 MVP 范围内。
- 调试人员不得直接访问生产 P3 数据。

后续扩展需支持：

- 机构管理员。
- 督导。
- 个案转介。
- 分级授权。
- 数据导出审批。

## 11. 数据生命周期

| 阶段 | 要求 |
| --- | --- |
| 创建 | 告知用途，尽量少收集 |
| 使用 | 仅用于记录、整理、总结、风险提示、报告、提醒 |
| 存储 | 按分级采取缓存、加密、访问控制 |
| 导出 | 用户主动触发，标记敏感文件 |
| 删除 | 正式上线前补齐删除和恢复策略 |
| 备份 | 生产环境备份同样按 P3 数据保护 |

## 12. 验收 Checklist

- [ ] 所有 P3 字段不进入普通日志。
- [ ] token 使用安全存储。
- [ ] 录音临时文件保存在 App 私有目录，上传失败可恢复或重试。
- [ ] 音频对象存储不使用公开 URL。
- [ ] 转写任务日志不包含转写正文。
- [ ] 未登录不能访问个案和记录。
- [ ] API 对个案、记录、报告、提醒做归属校验。
- [ ] AI 调用记录 prompt/model/schema 版本。
- [ ] AI 输入输出不在错误日志中明文出现。
- [ ] 系统通知默认不展示敏感内容。
- [ ] 退出登录后清理本地敏感缓存。
- [ ] 风险提示必须有人工作业状态。
- [ ] 报告草稿能追溯来源记录。
