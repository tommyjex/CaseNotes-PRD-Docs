# Phase 0：需求澄清与边界定义交付总览

## 1. 交付目标

本目录沉淀心理咨询个案助手 App 的 Phase 0 产物，作为后续 Flutter 页面设计与客户端开发、后端 API 设计、AI 工作流设计和安全合规评审的输入。

已沿用的项目方向：

- 阶段目标：MVP 验证。
- 产品形态：心理咨询个案助手手机 App。
- 端侧范围：Android + iOS。
- 技术方向：Flutter + flutter_easy_ui。
- 设计协作：弃用 Figma，直接基于 Flutter + flutter_easy_ui 完成设计和落地。
- 数据策略：云端优先。
- 核心入口：App 内录音、上传已有录音文件、云端 API 转写。
- AI 定位：基于转写/人工文本辅助整理、总结、风险提示和报告草稿，必须人工确认。

## 3. MVP 核心闭环

```text
登录 -> 创建个案 -> 录音/上传音频 -> 云端转写 -> 编辑确认文本 -> AI 整理/总结 -> 风险提示确认 -> 生成报告草稿 -> 创建跟进提醒
```

## 4. 阶段 0 结论

MVP 应聚焦单咨询师工作流，不进入机构协作和复杂后台能力。第一阶段优先验证录音/上传、云端转写、AI 辅助整理、风险提示、报告草稿和跟进提醒是否能显著减少咨询师文书工作，并保持人工专业判断和隐私保护边界。

## 5. 下一阶段建议

建议进入 Phase 1 前补齐以下决策：

1. 登录方式：手机号、邮箱或企业账号。
2. 默认咨询记录模板：SOAP、DAP、BIRP 或自定义。
3. AI Provider：内部模型、火山方舟、OpenAI/Claude，或多供应商可插拔。
4. 录音转写 API：云端异步转写供应商、支持格式、单文件时长限制和计费方式。
5. 音频保留策略：原始录音长期保留、用户可删除，还是转写完成后自动清理。
6. 报告导出范围：MVP 是否必须支持 PDF。
7. 提醒方式：仅 App 内提醒，还是同步支持系统推送。
8. 试点范围：内部试点、特定咨询师小组，还是准备公开上架。
