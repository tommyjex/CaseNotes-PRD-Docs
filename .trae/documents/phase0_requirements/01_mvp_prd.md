| 字段 | 内容 |
| --- | --- |
| 产品名称 | 心理咨询个案助手 App |

## 2. 背景与问题

心理咨询师在个案工作中需要持续记录咨询过程、整理来访者变化、形成阶段性总结、识别潜在风险并安排跟进。传统方式通常依赖手工笔记、文档模板或表格，存在以下问题：

- 记录分散，检索和追踪困难。
- 咨询后整理耗时，影响工作效率。
- 阶段性总结和报告需要重复整理历史记录。
- 风险线索可能散落在多次记录中，缺少及时提示。
- 跟进提醒依赖个人习惯，容易遗漏。

本产品希望通过移动端记录 + 云端同步 + AI 辅助整理，帮助咨询师更高效地完成个案管理，但 AI 输出只作为辅助草稿和风险提示，不替代专业判断。

## 3. 产品目标

### 3.1 MVP 目标

在 8 周左右完成可试点版本，使咨询师能够完成以下闭环：

1. 创建并管理个案。
2. 记录一次咨询内容。
3. 在 App 内录音，或上传已录制好的音频文件。
4. 调用云端 API 完成录音转写，生成可编辑转写文本。
5. 将转写文本或人工文本整理为结构化咨询记录。
6. 生成本次摘要、风险提示和报告草稿。
7. 咨询师人工确认或编辑 AI 输出。
8. 基于记录或 AI 建议创建跟进提醒。

### 3.2 不做目标

MVP 阶段暂不覆盖：

- 机构级多角色管理。
- 督导协作。
- 实时逐字转写和会中字幕，MVP 先支持录音结束后上传转写。
- 大规模量表库。
- 付费订阅。
- 电子签名。
- 自动诊断或自动危机干预。
- Web 管理后台。

## 4. 目标用户与使用场景

### 4.1 目标用户

| 用户 | 描述 | MVP 关注点 |
| --- | --- | --- |
| 个体心理咨询师 | 独立执业或机构内咨询师 | 快速记录、整理、查询、生成报告和提醒 |
| 试点运营人员 | 负责组织试点和收集反馈 | 观察核心功能是否提升效率 |
| 后续扩展用户 | 督导、机构管理者 | MVP 仅预留数据结构和权限扩展空间 |

### 4.2 典型场景

- 咨询过程中或咨询结束后，咨询师在手机上录音，或上传已录制好的音频文件。
- 系统将录音上传到云端转写 API，生成可编辑的咨询转写文本。
- 咨询师将零散记录整理成 SOAP/DAP 风格摘要。
- 咨询师查看 AI 给出的风险提示，并人工判断是否采纳。
- 咨询师为高风险或需回访个案创建跟进提醒。
- 咨询师在阶段性复盘前生成报告草稿，再人工编辑。

## 5. 核心原则

1. **专业人员主导**：所有 AI 输出均为草稿或提示，必须由咨询师确认后才能成为正式记录。
2. **隐私优先**：咨询记录、个案信息、风险提示默认按敏感数据处理。
3. **MVP 收敛**：第一阶段只完成核心闭环，不扩散到机构协作和复杂运营。
4. **结构化优先**：在自由文本之外保留关键字段，支撑检索、总结、报告和风险提示。
5. **可追溯**：AI 任务、输入快照、输出结果、确认状态都要留痕。

## 6. MVP 功能范围

### 6.1 账号与登录

#### 用户故事

作为咨询师，我希望用账号登录 App，以便安全访问自己的个案资料。

#### 功能需求

- 支持手机号或邮箱登录，MVP 可先实现一种。
- 支持登录态保持。
- 支持退出登录。
- 支持基础个人资料：姓名、机构名称、执业角色。

#### 验收标准

- 未登录用户不能访问个案数据。
- 登录过期后需要重新登录。
- token 不应明文写入普通本地存储。

### 6.2 个案管理

#### 用户故事

作为咨询师，我希望创建和管理个案，以便持续记录同一来访者的咨询过程。

#### 功能需求

- 创建个案：昵称或编号、性别、年龄段、主诉、标签、备注。
- 查看个案列表：支持按最近咨询时间排序。
- 搜索个案：支持昵称/编号/标签搜索。
- 个案详情：展示基础信息、咨询记录、风险提示、报告、提醒。
- 归档个案：归档后默认不出现在活跃列表。

#### 验收标准

- 创建个案不强制填写真实姓名。
- 列表可区分活跃和归档状态。
- 删除能力 MVP 可暂缓，避免误删敏感数据。

### 6.3 录音、上传与转写

#### 用户故事

作为咨询师，我希望在 App 内录制咨询音频，或上传已经录制好的音频文件，并由云端 API 自动转写，以减少手工输入成本。

#### 功能需求

- App 内录音：支持开始、暂停、继续、结束、放弃录音。
- 录音前提示：需确认已按专业伦理和适用法规取得必要授权或知情同意。
- 上传录音文件：支持从本地文件选择已录制好的音频文件。
- 音频格式：MVP 建议支持 `m4a`、`mp3`、`wav`，最终以云端转写 API 支持为准。
- 音频上传：展示上传进度、失败重试、取消上传。
- 云端转写：上传成功后创建转写任务，支持处理中、成功、失败状态。
- 转写文本编辑：咨询师可查看、编辑、删除或重新转写。
- 转写结果关联咨询记录：转写文本作为后续 AI 整理、总结、风险提示和报告草稿的主要输入。

#### 验收标准

- 录音文件和转写文本均按 P3 高敏数据处理。
- 录音中断、App 切后台、网络失败时，应尽量保留本地临时文件并给出明确恢复/重试入口。
- 未完成转写时不能直接触发基于转写文本的 AI 总结，但可以允许咨询师手动补充文本。
- 上传和转写失败必须提供重试或改为手动输入的路径。
- 系统通知和普通日志不得展示音频内容、转写原文或敏感片段。

### 6.4 咨询记录

#### 用户故事

作为咨询师，我希望基于录音转写文本快速形成咨询记录，并补充结构化字段，以便后续整理和报告。

#### 功能需求

- 新建咨询记录：关联个案、咨询日期、咨询时长、咨询方式。
- 原始记录：优先来自录音转写文本，同时支持手动长文本输入和编辑。
- 结构化字段：主诉变化、情绪状态、干预方式、本次目标、下次计划。
- 标签：支持自定义或预置标签。
- 草稿保存：未提交前可保存为草稿。
- 编辑记录：允许修改人工输入内容。
- 音频关联：记录可关联一个或多个音频文件及对应转写任务。

#### 验收标准

- 原始记录或转写文本为空时不能触发 AI 总结。
- 草稿不应丢失。
- 记录详情能展示人工内容与 AI 输出的区别。
- 记录详情能展示音频、转写文本、人工编辑文本和 AI 输出之间的关系。

### 6.5 AI 整理与总结

#### 用户故事

作为咨询师，我希望 AI 帮我把原始记录整理成结构化摘要，以减少咨询后的文书时间。

#### 功能需求

- 从咨询记录触发 AI 整理，主要输入为已确认或已编辑的转写文本。
- 支持生成结构化记录，默认模板为 SOAP 或 DAP，MVP 可先选一种。
- 支持生成本次咨询摘要。
- 支持生成跟进建议。
- 支持重新生成。
- 支持人工编辑并确认。

#### 验收标准

- AI 输出前显示“仅供专业参考，需人工确认”。
- AI 生成结果默认是草稿状态。
- 咨询师确认后，结果状态变为“已确认”。
- 每次生成应保存模型、prompt 版本、生成时间和确认状态。

### 6.6 风险提示

#### 用户故事

作为咨询师，我希望系统提示潜在风险线索，以帮助我更及时地关注高风险个案。

#### 功能需求

- 对咨询记录或转写文本生成风险提示。
- 风险等级：无明显风险、低风险、中风险、高风险、紧急关注。
- 风险类型：自伤/自杀、伤人风险、虐待或被虐待、严重情绪危机、其他。
- 展示证据片段：引用触发提示的原始记录片段。
- 咨询师必须选择确认、修改等级、标记误报或创建跟进。
- 高风险及以上默认建议创建跟进提醒。

#### 验收标准

- 风险提示不能显示为诊断结论。
- 风险等级必须带免责声明。
- 未经人工确认的风险提示不能进入正式报告。

### 6.7 报告草稿

#### 用户故事

作为咨询师，我希望基于历史记录生成报告草稿，以便更快完成阶段性复盘。

#### 功能需求

- 从个案详情发起报告生成。
- 选择报告范围：最近一次、最近 N 次、指定日期范围。
- 生成报告草稿：基本信息、咨询概况、变化趋势、风险关注、后续建议。
- 支持人工编辑报告草稿。
- 支持导出 PDF，MVP 可先只支持预览和复制，PDF 放入后续开发。

#### 验收标准

- 报告中必须标明 AI 生成内容需人工审核。
- 未确认的风险提示不能自动写入正式报告。
- 报告草稿应能追溯来源记录。

### 6.8 跟进提醒

#### 用户故事

作为咨询师，我希望从记录或风险提示创建提醒，以避免遗漏后续跟进。

#### 功能需求

- 手动创建提醒：标题、关联个案、提醒时间、备注。
- 从 AI 建议创建提醒。
- 从中高风险提示创建提醒。
- 提醒列表：待处理、已完成。
- 提醒通知：MVP 可先做 App 内提醒，系统推送可作为增强项。

#### 验收标准

- 过期提醒有明显标识。
- 完成提醒后保留记录。
- 提醒内容不应在锁屏通知中展示敏感详情，除非用户明确开启。

## 8. 数据与隐私要求

- 所有 API 必须使用 HTTPS。
- 账号 token 使用安全存储。
- 录音文件、转写文本、咨询记录、个案资料、风险提示、报告属于敏感数据。
- 后端日志不得记录完整咨询原文。
- AI 任务输入应做最小必要原则处理。
- 生产环境应支持审计日志。
- 数据导出、删除、备份和恢复流程需在正式上线前补齐。

详见：

- `04_data_classification_privacy.md`

## 9. 风险提示与人工确认要求

- AI 风险提示只做辅助提醒，不构成诊断、评估结论或危机干预指令。
- 风险提示必须展示证据片段和不确定性说明。
- 咨询师必须人工确认或标记误报。
- 高风险及紧急关注应引导咨询师按专业规范处理。

详见：

- `05_risk_disclaimer_confirmation_flow.md`

## 10. 成功指标

### 10.1 效率指标

- 咨询师能在 3 分钟内完成一次记录初稿。
- AI 结构化整理生成时间不超过 30 秒，允许异步处理。
- 报告草稿可减少至少 30% 的人工整理时间。

### 10.2 质量指标

- AI 输出 JSON schema 成功率 >= 95%。
- 咨询师对摘要“可用或稍改可用”的评价 >= 70%。
- 风险提示必须能解释触发依据。

### 10.3 安全指标

- 未登录状态无法访问任何个案数据。
- 敏感数据不出现在客户端普通日志和后端普通日志。
- AI 输出未经确认不得进入正式报告。

## 11. 验收标准

MVP 通过验收需满足：

- 账号、个案、录音/上传、云端转写、咨询记录、AI 整理、风险提示、报告草稿、提醒形成完整闭环。
- Android 和 iOS 均可完成核心流程。
- Flutter 静态页面和 flutter_easy_ui 组件样例覆盖关键页面。
- 数据分类和隐私分级表已作为研发约束。
- 风险提示免责声明和人工确认流程已进入 UI 与接口设计。

## 12. 待确认事项

| 问题 | 默认假设 | 影响 |
| --- | --- | --- |
| 登录方式 | MVP 先选手机号或邮箱之一 | 影响认证接口和注册流程 |
| 默认记录模板 | 先选 SOAP 或 DAP 之一 | 影响 AI schema 和 UI 字段 |
| AI Provider | 先按可插拔 Provider 设计 | 影响后端适配和成本 |
| 录音转写 API | 先按云端异步转写任务设计 | 影响音频上传、任务轮询、成本和隐私协议 |
| 音频保留策略 | MVP 默认服务端保存原始音频，后续可配置自动删除 | 影响对象存储、审计和用户信任 |
| 报告导出 | MVP 可先预览/复制，PDF 延后 | 影响对象存储和导出任务 |
| 系统推送 | MVP 可先 App 内提醒 | 影响 iOS/Android 推送配置 |
| 目标市场 | 默认内部/小范围试点 | 影响合规、隐私协议和上架要求 |
